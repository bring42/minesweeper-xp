# Minesweeper XP — in-browser via v86

Runs the original **Windows XP `winmine.exe`** in the browser using the
[v86](https://github.com/copy/v86) x86 emulator, deployed as a static site on
**Cloudflare Pages** via **GitHub Actions** — no R2, no server.

## How it works

```
browser
  └── index.html  →  loads v86.js + v86.wasm
                  →  registers sw.js (Service Worker)
                  →  boots disk.img (Windows 98 + minesweeper)

sw.js            intercepts GET /disk.img Range requests
                  →  fetches /chunks/disk.img.0000 … .NNNN in parallel
                  →  reassembles the byte range and returns it
                  →  caches each chunk with Cache API (no re-download)

Cloudflare Pages  serves everything as static files
                  (each chunk ≤ 10 MiB, well under the 25 MiB/file limit)
```

v86 uses HTTP Range requests (`async: true`) so only the sectors it actually
needs are fetched on first load — typically 5–15 MB for boot + minesweeper.

## One-time setup

### 1. Prerequisites

| Tool | Install |
|------|---------|
| QEMU | `brew install qemu` |
| Python 3 | pre-installed on macOS |
| curl | pre-installed on macOS |
| [GitHub CLI](https://cli.github.com) | `brew install gh` |
| [Wrangler](https://developers.cloudflare.com/workers/wrangler/) | `npm i -g wrangler` (optional, Actions does it) |

### 2. Build the disk image

You need a **Windows 98 SE ISO**.  If you own a licence, rip the CD.
The Internet Archive also hosts preservation copies:
<https://archive.org/search?query=windows+98+SE>

```bash
bash scripts/build-image.sh /path/to/win98se.iso
```

The script will:
1. Download `winmine.exe` from archive.org automatically
2. Create a blank `disk.img` (256 MiB)
3. Launch QEMU so you can install Windows interactively (~10 min)
4. Copy `winmine.exe` into the image and configure auto-launch

> **Tip — faster path:** If you already have a working v86-compatible
> Windows 9x/Me/2000/XP disk image, skip to step 3.

### 3. Chunk the disk image

```bash
python3 scripts/chunk.py disk.img
```

This writes `public/chunks/disk.img.0000`, `.0001`, … and `meta.json`.

### 4. Download emulator files

```bash
bash scripts/prepare.sh
```

Downloads `v86.js`, `v86.wasm`, `seabios.bin`, `vgabios.bin` into `public/`.

### 5. Test locally

```bash
cd public
python3 -m http.server 8080
# open http://localhost:8080
```

> Service Workers require HTTPS **or** `localhost`.  Plain `python3 -m http.server`
> on localhost works fine.

## Deploy

### Create a GitHub repo

```bash
gh repo create minesweeper-xp --public --source=. --remote=origin --push
```

### Upload disk.img as a GitHub Release asset

```bash
gh release create v1.0 disk.img \
  --title "Disk image v1.0" \
  --notes "Initial Windows 98 + winmine.exe image"
```

> The disk image stays out of Git history this way.  GitHub Actions
> downloads it fresh on every deploy.

### Add Cloudflare secrets to GitHub

1. Log in to [Cloudflare](https://dash.cloudflare.com) and create a
   **Pages project** named `minesweeper-xp` (or adjust the name in
   `.github/workflows/deploy.yml`).
2. Create an API token with the *Cloudflare Pages: Edit* permission.
3. In your GitHub repo → Settings → Secrets → Actions, add:

| Secret | Value |
|--------|-------|
| `CLOUDFLARE_API_TOKEN` | the token from step 2 |
| `CLOUDFLARE_ACCOUNT_ID` | your Cloudflare Account ID |

### Push to trigger deploy

```bash
git push origin main
```

GitHub Actions will:
1. Run `scripts/prepare.sh` to download v86 binaries
2. Download `disk.img` from your GitHub Release
3. Run `scripts/chunk.py` to create `public/chunks/`
4. Deploy the whole `public/` directory to Cloudflare Pages

Your site will be live at `https://minesweeper-xp.pages.dev` (or your
custom domain).

## Project structure

```
.
├── public/
│   ├── index.html        ← main page (loads v86, handles loading UI)
│   ├── sw.js             ← Service Worker (chunk reassembly + caching)
│   ├── _headers          ← Cloudflare Pages response headers
│   ├── v86.js            ← generated by prepare.sh (gitignored)
│   ├── v86.wasm          ← generated by prepare.sh (gitignored)
│   ├── seabios.bin       ← generated by prepare.sh (gitignored)
│   ├── vgabios.bin       ← generated by prepare.sh (gitignored)
│   └── chunks/           ← generated by chunk.py (gitignored)
│       ├── disk.img.0000
│       ├── disk.img.0001
│       └── meta.json
├── scripts/
│   ├── prepare.sh        ← downloads v86 + BIOS files
│   ├── chunk.py          ← splits disk.img into chunks
│   └── build-image.sh    ← builds Windows disk image via QEMU
├── minesweeperxp/        ← original archive.org metadata
├── .github/workflows/
│   └── deploy.yml        ← GitHub Actions CI/CD
└── .gitignore
```

## Notes

- **Disk image size:** ~256 MiB virtual, typically 50–120 MiB of real data.
  With `async: true` v86 only fetches the sectors it needs — first boot
  usually transfers ≤ 20 MB.
- **Chunk caching:** `sw.js` caches each chunk in the browser's Cache API.
  Repeat visits load instantly from cache.
- **No R2, no Workers, no server** — pure static hosting.
- **Windows licence:** Ensure you hold a valid licence for the Windows image
  you use.
